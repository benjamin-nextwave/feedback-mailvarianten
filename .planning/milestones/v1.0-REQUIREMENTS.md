## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Database

- [x] **DB-01**: Supabase schema with forms table (id, client_name, slug, status, webhook_url, created_at, updated_at)
- [x] **DB-02**: Supabase schema with email_variants table (id, form_id FK, email_type, variant_number, subject_line, email_body, sort_order, created_at)
- [x] **DB-03**: Supabase schema with feedback_responses table (id, form_id FK, variant_id FK, feedback_text, submitted_at)
- [x] **DB-04**: RLS policies allowing authenticated users full access for dashboard operations
- [x] **DB-05**: RLS policies allowing anonymous SELECT on forms and email_variants filtered by slug
- [x] **DB-06**: RLS policy allowing anonymous INSERT on feedback_responses
- [x] **DB-07**: Migration SQL file(s) in /supabase/migrations/

### Dashboard

- [x] **DASH-01**: Dashboard home page listing all forms in a table with columns: Klantnaam, Aangemaakt op, Status badge (Openstaand/Ingeleverd), Deelbare link, Acties
- [x] **DASH-02**: Clickable/copy-able public URL for each form in the list
- [x] **DASH-03**: Form creation page with Klantnaam (required), Webhook URL (optional), and Eerste mail variants (1-5)
- [x] **DASH-04**: Optional Opvolgmail 1 toggle on creation page with variant count and subject/body fields
- [x] **DASH-05**: Optional Opvolgmail 2 toggle on creation page (only if Opvolgmail 1 enabled) with variant count and subject/body fields
- [x] **DASH-06**: Auto-generated human-friendly slug from client name + short random string on form creation
- [x] **DASH-07**: Form detail page showing client name, creation date, status, and shareable link
- [x] **DASH-08**: Form detail page showing all email variants grouped by type (Eerste mail, Opvolgmail 1, Opvolgmail 2)
- [x] **DASH-09**: Form detail page showing feedback text in green-highlighted card per variant, or gray "Nog geen feedback ontvangen" placeholder
- [x] **DASH-10**: Delete form action with Dutch confirmation modal

### Public Form

- [x] **FORM-01**: Public feedback page at /feedback/[slug] accessible without authentication
- [x] **FORM-02**: Header showing "Invulformulier voor [Klantnaam]" with NextWave Solutions text branding
- [x] **FORM-03**: Intro text "Hieronder vind je de e-mailvarianten die wij voor je hebben opgesteld. Laat per variant je feedback achter."
- [x] **FORM-04**: Email variants displayed in cards grouped by type with section headers (Eerste mail, Opvolgmail 1, Opvolgmail 2)
- [x] **FORM-05**: Each variant card shows "Variant [number]" label, subject line, and email body in email-style preview
- [x] **FORM-06**: Per-variant feedback textarea labeled "Jouw feedback" with placeholder "Schrijf hier je feedback over deze variant..."
- [x] **FORM-07**: Submit button "Feedback versturen" at the bottom
- [x] **FORM-08**: Validation requiring at least one feedback field filled with Dutch error message
- [x] **FORM-09**: Save all non-empty feedback entries to feedback_responses and update form status to 'completed' on submit
- [x] **FORM-10**: Fire-and-forget webhook POST with JSON payload (form_id, client_name, slug, submitted_at, feedback_count, responses array)
- [x] **FORM-11**: Success message "Bedankt! Je feedback is succesvol verstuurd." after submission
- [x] **FORM-12**: Read-only view showing previously submitted feedback when client revisits (no re-submission)

### Design

- [x] **UI-01**: Light professional theme — white (#FFFFFF) / light gray (#F9FAFB) backgrounds, blue primary (#2563EB), green success (#16A34A), amber pending (#F59E0B)
- [x] **UI-02**: All UI text, labels, buttons, placeholders, and messages in Dutch (Netherlands)
- [x] **UI-03**: Cards with subtle shadows and rounded corners
- [x] **UI-04**: Clean typography with adequate spacing and visual hierarchy

### Infrastructure

- [x] **INFRA-01**: .env.example file with NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY
- [x] **INFRA-02**: Deployable to Vercel with zero additional configuration beyond environment variables
