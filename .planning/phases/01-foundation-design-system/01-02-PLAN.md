  <action>
    1. Initialize shadcn/ui using `npx shadcn@latest init`. When prompted or via flags, choose:
       - Style: "new-york" (more polished, better for professional apps)
       - Base color: "neutral" (we will customize colors)
       - CSS variables: yes
       - Import alias: "@/*" (matches tsconfig)
       If the CLI requires interactive input, use `npx shadcn@latest init -d` for defaults, then adjust components.json manually. The key is that `components.json` exists and points to `src/components/ui` for component output.

    2. After shadcn/ui init, customize the CSS custom properties in `src/app/globals.css` to implement the project's design system colors (UI-01):

       In the `:root` (light mode) section, set:
       - `--primary`: HSL value for #2563EB (blue) -- approximately `217.2 91.2% 59.8%`
       - `--primary-foreground`: white `0 0% 100%`
       - `--background`: #FFFFFF -- `0 0% 100%`
       - `--muted`: #F9FAFB -- approximately `210 40% 98%`
       - `--muted-foreground`: #6B7280 -- approximately `215.4 16.3% 46.9%`
       - `--card`: #FFFFFF -- `0 0% 100%`
       - `--card-foreground`: dark text for readability

       Add custom CSS variables for status colors (not part of shadcn defaults):
       - `--success`: HSL for #16A34A (green) -- approximately `142.1 76.2% 36.3%`
       - `--success-foreground`: `0 0% 100%`
       - `--pending`: HSL for #F59E0B (amber) -- approximately `37.7 92.1% 50.2%`
       - `--pending-foreground`: `0 0% 0%`

       Keep the light theme ONLY. Remove or comment out the `.dark` theme section if shadcn/ui generates one (UI-01 specifies light theme only).

       Also set `--radius` to `0.5rem` for subtle rounded corners (UI-03).

    3. Update `src/app/layout.tsx`: Import the Inter font from `next/font/google` (clean, professional). Apply font class to body. Ensure body has `className="min-h-screen bg-background font-sans antialiased"`.

    4. Add core shadcn/ui components one by one:
       ```
       npx shadcn@latest add button
       npx shadcn@latest add card
       npx shadcn@latest add input
       npx shadcn@latest add textarea
       npx shadcn@latest add badge
       npx shadcn@latest add label
       ```

       These are the minimum components needed for the feedback platform MVP. They will be installed to `src/components/ui/`.

    5. After installing badge component, modify `src/components/ui/badge.tsx` to add two custom variants for status badges:
       - `success` variant: uses `--success` and `--success-foreground` CSS variables for background and text
       - `pending` variant: uses `--pending` and `--pending-foreground` CSS variables for background and text

       Add these to the existing badgeVariants alongside the default, secondary, destructive, and outline variants.

    6. Verify all components import correctly by running `npm run build`.
  </action>
